How to Build this Website

January 16, 2021 - 3 minutes read - 483 words

There are many options out there to build and host a website with a custom domain. The steps listed below are how I created this website by building with Hugo and hosting on Github pages with a custom domain from Google.

Instructions

  1. Install Go. Follow the instructions on the page to unpack, export to PATH, and check the go version to ensure it’s been installed correctly.

  2. Install Hugo. I chose to install from source.

  3. Export all required paths to PATH.

export PATH=$PATH:/snap/bin
export PATH=$PATH:/snap/bin/hugo
export PATH=$PATH:/usr/local/go_install/go/bin

  1. Choose a theme to your liking. You can check the tags to filter by your requirements.

  2. Buy a domain from Google.

  3. Follow instructions on the Hugo quickstart page. 

  4. Copy config.toml from exampleSite (themes -> ananke -> exampleSite) to quickstart 

  5. Copy the contents of the entire layouts folder from exampleSite to quickstart

  6. Add sections manually by creating new folders (with exampleSite as reference). You can customize config.toml.

hugo server -D

This allows you to see a local version of the site and keep iterating before publishing changes. Sometimes, you might have to terminate this process in the terminal (Ctrl+C) and rerun this command to refresh changes but most changes are instantly visible.

  1. Customize menu order for sections by modifying config.toml and modifying weights. Here’s an example.

  2. Customize menu order for individual pages by adding the weight parameter in index.md file.

  3. Render website in public folder.

cd quickstart
hugo

  1. Create a github.io page: create a new repo called .github.io

  2. Add contents of rendered website to this github repo:

cd public
git init
git add .
git remote add origin https://github.com/username/username.github.io.git
git commit -m “first commit”
git push origin master

You could also put this in a script. 

  1. Congratulations! .github.io is now published!

  2. Now add the custom domain by modifying Settings on the github repo.

  3. Add a CNAME record to Google Domains to link the custom domain to the github.io page. Then, configure an ‘apex domain,' by creating records under A with your DNS provider. Here’s an example with screenshots.

  4. To verify that your Github page redirects to your custom domain, see Section 6.

  5. Here’s a resource for troubleshooting custom domains.

  6. You can also enforce HTTPS.

  7. If you make any changes locally and you want to update them to your published site, just go through the following steps:

cd quickstart
hugo
cd public
git add .
git commit -m "some changes"
git push 		# push all changes to the remote
  1. Note - to push local changes on top of changes made directly to the github repo, use git pull –rebase

Specifics about Content and Formatting

  • config.toml allows you to set pagination ie number of featured previews.
  • Customize fonts.
  • Add images to static -> images folder
  • Set absolute URLs for social media icons by directly substituting the href parameter (otherwise they are relative to baseURL) in layouts->partials->social-follow.html
  • Changing text colour.
  • A handy Markdown guide.
